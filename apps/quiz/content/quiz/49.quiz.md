::question{:type="questionType"}
In the code snippet below, `name` and `doubleCount` lose their reactivity due to destructuring from the pinia store. How can you fix this?

<br />

```js
import { useCounterStore } from '@/stores/counter'

const store = useCounterStore()
const { name, doubleCount } = store
```

::

::explanation{:type="explanationType"}

**Explanation: To extract properties from the store while keeping its reactivity, you need to use storeToRefs()**

<br />

```js
<script setup>
import { useCounterStore } from '@/stores/counter'
import { storeToRefs } from 'pinia'

const store = useCounterStore()
const { name, doubleCount } = storeToRefs(store)
</script>
```

::

::vueTip{:type="tip" :reference="https://pinia.vuejs.org/core-concepts/#Destructuring-from-a-Store" :position="49"}

#text

Use storeToRefs to destructure reactive properties from a Pinia store while preserving reactivity.

#code

```js
import { storeToRefs } from 'pinia'

const store = useCounterStore()
const { name, doubleCount } = storeToRefs(store)
```
