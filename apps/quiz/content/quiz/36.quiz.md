::question{:type="questionType"}
By default, route paths are case-insensitive. Hence /users will also match /Users and /USERS. Which route option makes route paths case sensitive?

::

::explanation{:type="explanationType"}

**Explanation: { path: '/users', 'sensitive': true } makes route paths case sensitive.**

<br />
This means that /users will match /users but will not match /Users or /USERS.

```js
const routes = [
  { path: '/users', sensitive: true },
]
```
