::question{:type="questionType"}
What is potentially problematic about the code snippet below?

<br />

```js
<div v-html="userDefinedContent"></div>
```

::

::explanation{:type="explanationType"}

**if you allow unsanitized, user-provided content to be executed (as either HTML, JavaScript, or even CSS), you might open yourself up to attacks.**

<br />

Libraries like [DOMPurify](https://www.npmjs.com/package/dompurify) can help sanitize HTML content before rendering it with v-html. However, sanitization should also be done server-side to ensure maximum security.

::

::vueTip{:type="tip" :reference="https://vuejs.org/guide/best-practices/security.html#rule-no-1-never-use-non-trusted-templates" :position="23"}

#text

Always sanitize user-provided content before using v-html to prevent XSS attacks; use libraries like DOMPurify and validate server-side.

#code

```js
import DOMPurify from 'dompurify';

const sanitizedContent = DOMPurify.sanitize(userDefinedContent);

<div v-html="sanitizedContent"></div>
```
